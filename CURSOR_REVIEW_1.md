## P0 — fix the event proxy

The fixture and schema use normalized image coordinates where Y increases downward. The current `stepEvents.ts` treats a local Y minimum as contact, which identifies the lifted heel rather than the low heel. Seeded noise on the flat portion also creates 65 accepted events in 12 seconds, so the displayed values are not a credible proxy.

Replace the detector with a deterministic **heel-low event proxy**:

1. Smooth each heel-Y series with a small centered moving average.
2. Find the upward crossing into the low-heel band (large image Y), using a per-side threshold derived from that fixture's smoothed range. A threshold around `min + 0.75–0.85 * (max-min)` is appropriate for the synthetic fixture.
3. Use a refractory period around 500ms so one stance plateau cannot create multiple events.
4. Keep the confidence window and frame-gap rejection.
5. Do not call the event a measured heel strike or ground contact. In UI/docs use `heel-low event estimate` or `accepted heel-low events`.

Add a fixture-specific test that the accepted fixture has a sane deterministic count (roughly one event per synthetic cycle, not dozens) and same-side median intervals in the range generated by the fixture's 0.95Hz and 0.88Hz signals. These are fixture invariants, not population normality thresholds.

Rename visible `step interval` labels to `same-side event interval`, because the implementation computes consecutive events for the same side. Update README/MANIFEST descriptions from heel minima/contact detection to the accurate proxy wording.
